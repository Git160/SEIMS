1 模块开发Demo
2 -------------
3 
4 以一个简单Demo为例介绍，模块开发的整体流程。
5 
6 一般来说，模块编写基本步骤如下：
7 
8 + 1. 阅读相关文献和模型文档，编写模型文档的理论部分；
9 + 2. 设计模块名，基本命名规则为`过程_算法`，如`PET_H`为蒸散发过程（PET）的Hargreaves方法实现；缩写规范自行查找，比如`YLD`可以是作物产量`Yield`的缩写，没有指定具体算法的，命名只有`过程`也可；
10 + 3. 根据模型的相关理论，参考已有成熟模型的代码，列出模型的输入（包括以其他模块的输出作为输入）、输出，编写元数据函数(`MetadataInformation`)代码，即`api.cpp`；
11 + 在头文件中定义所需变量，并在实现文件(`.cpp`)中做参数初始化（构造函数）、赋值（`SetValue`, `Set1DData`, `Set2DData`系列函数）和有效性检查（`CheckInputData`系列函数）的工作；
12 + 根据公式，编写`Execute`函数的代码，实现模拟功能
13 + 编写输出代码，`Get1DData`，`Get2DData`等
14 + 代码检查、测试
15

30 ## 1. 新建模块
31 
32 这里，默认你已经掌握SEIMS代码基本结构和`CMAKE`编译知识，并已经成功编译出SEIMS的VS工程，如果没有，请往前阅读相关文档。
33 
34 **模块功能设计**：
35 
36 + 测试模型对单层栅格和多层栅格数据的支持
37 + 分别读取Raster1D和Raster2D数据，做简单运算之后，输出
38 + 模块命名为`IO_TEST`
39 
40 **利用CMAKE新建模块工程**：
41

51 ## 2. 编写元数据函数
52 
53 为了便于统一修改，并保持SEIMS整体的一致性，`api.cpp`中所有字符串类型的元数据信息均在`src/base/util/text.h`头文件中以宏定义`#define`的形式进行管理。
54 
55 ### 2.1 SetClass
56 定义模块所属模块组及描述，在`text.h`中定义为`MCLS_`及`MCLSDESC_`，如：
57 
58 ```cpp
60 #define MCLS_PET "Potential Evapotranspiration"
61 #define MCLSDESC_PET "Calculate the potential evapotranspiration"
62 ```
63 
64 ### 2.2 SetID, SetDescription
65 定义该模块的唯一标识符ID和描述，在`text.h`中定义为`MID_`及`MDESC_`，如：
66 
67 ```cpp
68 #define MID_PET_H "PET_H"
69 #define MDESC_PET_H "Hargreaves method for calculating the potential evapotranspiration."
70 ```
71 SetName 同 SetID
72 
73 ### 2.3 AddParameter
74 
75 所有从数据库中原始读取的参数，均以AddParameter形式定义，这些参数在模拟开始前从数据库中读取，以指针的形式在不同模块间传递，模拟开始后，不重复读取！
76 
77 共有5个输入：
78 
79 + 变量名，在`text.h`中定义为`VAR_`，如CN2值定义为`VAR_CN2`
80 + 单位，在`text.h`中定义为`UNIT_`，如秒的单位为`UNIT_SECOND`
81 + 描述，在`text.h`中定义为`DESC_`，如CN2值得描述为`DESC_CN2`
82 + 来源，来源共有如下几种：
83  + `File_Input`:来自模型配置文件`file.in`,如模拟时间步长
84  + `Source_HydroClimateDB`：来自水文气象数据库，如气温
85  + `Source_ParameterDB`：来自模型参数数据库，包括MongoDB中`Parameters`表和`spatial`表
86 + 数据类型，为`dimensionTypes`枚举型变量，主要包括：
87  + `DT_Single`：单个数值型
88  + `DT_Array1D`：一维数组，如站点平均气温
89  + `DT_Array2D`：二维数组，如栅格分层计算的层数及每层栅格的索引值
90  + `DT_Raster1D`：一维栅格数据，如DEM
91  + `DT_Raster2D`：二维栅格数据，如多层土壤属性栅格
92  + `DT_Scenario`：情景数据
93 
94 ### 2.4 AddInput
95 所有来源于其他模块输出的参数均以AddInput的形式定义，这部分参数在模块运行之前，检查其他模块是否有输出该参数，如果有，则获取指针；如果该参数尚未初始化，则由对应模块的`initialOutputs()`函数进行初始化并输出。
96 
97 与AddParameter的设置基本一致，唯一的区别在于AddInput的变量来源必须是`Source_Module`。
98 
99 二者的区别为：
100 
101 + AddParameter中的参数，在模型构建时（即开始逐时间步长计算之前）一次性读取（除气象水文数据外），具体参见`src/base/ModuleFactory.cpp`中的：
102 
103 ```cpp
104 void ModuleFactory::SetData()
105 ```
106 
107 + 而AddInput的参数，在模型开始第一个时间步长运行前，获取该参数的内存地址：
108 
109 ```cpp
110 void ModuleFactory::GetValueFromDependencyModule()
111 ```
112 
113 ### 2.5 AddOutput
114 
115 AddOutput需要设置变量名、单位、描述、数据类型，不再赘述。
116 
117 
118 ### 2.6 Demo
119 
120 我们设定了测试模型输入输出的模块目的，这里我们可以这样定义AddParameter和AddOutput（暂不涉及从其他模块中读取变量）：读取一维栅格数据`VAR_CN2`和二维栅格数据`VAR_CONDUCT`，分别输出为`CN2_M`和`K_M`：
121 
122 ```cpp
124 mdi.AddParameter(VAR_CN2, UNIT_NON_DIM, DESC_CN2, Source_ParameterDB, DT_Raster1D);
125 mdi.AddParameter(VAR_CONDUCT, UNIT_WTRDLT_MMH, DESC_CONDUCT, Source_ParameterDB, DT_Raster2D);
127 mdi.AddOutput("CN2_M",UNIT_NON_DIM, DESC_CN2, DT_Raster1D);
128 mdi.AddOutput("K_M",UNIT_WTRDLT_MMH,DESC_CONDUCT,DT_Raster2D);
129 ```

177 ## 4. 实现函数编写
178 
179 接下来我们可以进入实现函数编写的阶段。
180 
181 ### 4.1 参数初始化
182 
183 按照AddParameter，AddInput，AddOutput中参数定义顺序，在构造函数中依次进行参数初始化，基本原则为：
184 
185 + `DT_Single`类型数据设置为-1或者`NODATA`(`utils.h`中定义)
186 + `DT_Array1D`, `DT_Raster1D`, `DT_Array2D`和`DT_Raster2D`类型变量设置为`NULL`
187 
188 这里我们这样初始化：
189 
190 ```cpp
191 IO_TEST::IO_TEST(void):m_nCells(-1), m_raster1D(NULL), m_raster2D(NULL),
192  m_output1Draster(NULL), m_output2Draster(NULL)
193 {
194 }
195 ```
196 
197 ### 4.2 内存释放（析构函数）
198 
199 基本原则是：凡是在该模块中new的内存均需要在析构函数中释放，除非该变量在其他地方被显式释放。对于一维数组利用`delete[]`即可，对于二维数组则需要遍历释放，并重新赋值为`NULL`避免“野指针”。
200 
201 这里我们这样释放内存：
202 
203 ```cpp
204 IO_TEST::~IO_TEST(void)
205 {
206  if (m_output1Draster != NULL)
207  delete[] m_output1Draster;
208  m_output1Draster = NULL;
209  if (m_output2Draster != NULL)
210  {
211  for(int i = 0; i < m_nCells; i++)
212  delete[] m_output2Draster[i];
213  delete[] m_output2Draster;
214  m_output2Draster = NULL;
215  }
216 }
217 ```
218 
219 由于释放内存频繁用到，因此，在`utils.h`中定义了模板函数用于释放内存：
220 
221 ```cpp
222 #include "utils.h"
223 if(m_output1Draster != NULL) Release1DArray(m_output1Draster);
224 if(m_output2Draster != NULL) Release2DArray(m_nCells, m_output2Draster)
225 ```
226 
227 ### 4.3 变量赋值
228 
229 赋值时首先进行数据长度检查（`CheckInputSize()`）,在进行长度检查时，顺便给栅格个数（`m_nCells`）赋值，随后对变量名进行匹配（`StringMatch()`），如果匹配则赋值，以`Set1DData`为例:
230 
231 ```cpp
232 void IO_TEST::Set1DData(const char* key, int n, float* data)
233 {
234  if(!this->CheckInputSize(key,n)) return;
235  string sk(key);
236  if(StringMatch(sk, VAR_CN2)) this->m_raster1D = data;
237 }
238 
239 bool IO_TEST::CheckInputSize(const char* key, int n)
240 {
241  if(n<=0)
242  throw ModelException("IO_TEST","CheckInputSize","Input data for "+string(key) +" is invalid. The size could not be less than zero.");
243  if(this->m_nCells != n)
244  {
245  if(this->m_nCells <=0) this->m_nCells = n;
246  else
247  throw ModelException("IO_TEST","CheckInputSize","Input data for "+string(key) +" is invalid. All the input data should have same size.");
248  }
249  return true;
250 }
251 ```
252 
253 ### 4.4 参数有效性检查
254 
255 有效性检查函数`CheckInputData()`用于逐个判断SetValue系列函数中设置的输入是否均以成功赋值，如没有，则抛出异常。
256 
257 ```cpp
258 bool IO_TEST::CheckInputData()
259 {
260  if(this->m_date == -1)
261  throw ModelException("IO_TEST","CheckInputData","You have not set the time.");
262  if(m_nCells <= 0)
263  throw ModelException("IO_TEST","CheckInputData","The dimension of the input data can not be less than zero.");
264  if(m_raster1D == NULL)
265  throw ModelException("IO_TEST","CheckInputData","The 1D raster input data can not be less than zero.");
266  if(m_raster2D == NULL)
267  throw ModelException("IO_TEST","CheckInputData","The 2D raster input data can not be less than zero.");
268  return true;
269 }
270 ```
271 
272 ### 4.5 编写Execute函数
273 
274 在Execute函数中，首先需要为输出变量开辟内存空间，并赋初值，然后进行模块功能的实现。
275 
276 这里我们以简单的算法为例：
277 
278 ```cpp
279 int IO_TEST::Execute()
280 {
282  if (m_output1Draster == NULL)
283  {
284  m_output1Draster = new float[m_nCells];
285 #pragma omp parallel for
286  for (int i = 0; i < m_nCells; ++i)
287  m_output1Draster[i] = 0.f;
288  }
289  if (m_output2Draster == NULL)
290  {
291  m_output2Draster = new float*[m_nCells];
292 #pragma omp parallel for
293  for (int i = 0; i < m_nCells; ++i)
294  {
295  m_output2Draster[i] = new float[m_soilLayers];
296  for(int j = 0; j < m_soilLayers; j++)
297  m_output2Draster[i][j] = 0.f;
298  }
299  }
301 #pragma omp parallel for
302  for (int i = 0; i < m_nCells; ++i)
303  {
304  m_output1Draster[i] = m_raster1D[i] * 0.5f;
305  for(int j = 0; j < m_soilLayers; j++)
306  m_output2Draster[i][j] = m_raster2D[i][j] + 2.f;
307  }
308  return true;
309 }
310 ```
311 
312 与内存释放类似，为方便参数初始化，在`util.h`中定义了模板函数，调用示例如下：
313 
314 ```cpp
316 if(m_output1Draster == NULL) Initialize1DArray(m_nCells, m_output1Draster, 0.f)
317 if(m_output2Draster == NULL) Initialize2DArray(m_nCells, m_soilLayers, m_output2Draster, 0.f)
318 ```
319

321 
322 与变量赋值代码类似，首先进行变量名匹配，然后进行赋值，以`Get2DData`为例：
