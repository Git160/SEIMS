10 # 1. Prerequisites
11 
12 SEIMS模型采用C++和Python编写，支持子流域-栅格双层并行计算，在macOS下推荐采用GCC进行编译安装，安装前的准备包括：
13 + GCC49 -- C++源码编译器
14 + OpenMPI -- 编译MPI并行程序，如TauDEM、SEIMS-MPI版本等
15 + CMake -- 管理代码编译
16 + GDAL 1.x with Python -- 矢栅数据读写库
17 + mongo-c-driver 1.3.5 或更高 -- NoSQL数据库MongoDB的驱动库
18

20 ## 1.1. GCC49
21 macOS中最便捷配置GCC版本的方式是通过[Homebrew](http://brew.sh/)自动安装。
22 + 在终端输入代码`/usr/bin/ruby -e "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/master/install)"
23 ` 进行安装`brew`工具
24 + 输入`brew tap homebrew/versions`更新`brew`中软件版本
25 + 输入`brew install gcc49`安装
26 + 安装大约需要60分钟，耐心等待安装完成，可通过`gcc-4.9 --version`查看版本信息：
27 
28 ```
29 gcc-4.9 --version
30 gcc-4.9 (Homebrew gcc49 4.9.3) 4.9.3
31 Copyright (C) 2015 Free Software Foundation, Inc.
32 ```

34 ## 1.2. OpenMPI
35 + 从[官网下载OpenMPI源码](https://www.open-mpi.org/software/ompi/v1.10/)，推荐安装1.x系列的最新版，目前最新的是1.10.4，解压至文件夹，如`/Users/zhulj/apps/openmpi-1.10.4
36 `
37 + 配置config，指定使用GCC-4.9编译，如下：`./configure CC=gcc-4.9 CXX=g++-4.9 --prefix=/usr/local`
38 + `make all`
39 + `sudo make install`
40 
41 安装完成之后，在终端中输入`mpic++ --showme`，会提示`g++-4.9 -I/usr/local/include -L/usr/local/lib -lmpi_cxx -lmpi`，则表明OpenMPI是由GCC49编译的。

43 ## 1.3. CMake
44 
45 CMake是一个跨平台的安装或编译工具，可以用简单的语句来描述安装或编译过程。CMake通过`CmakeLists.txt`文件能够输出各种各样的makefile或者IDE工程。
46 
47 CMake可以从其[官网免费下载](http://www.cmake.org/files)安装GUI版本，推荐安装3.0以上版本，也可通过`brew install cmake`安装。

49 ## 1.4. GDAL with Python
50 
51 SEIMS的矢栅数据读写基于`GDAL 1.x`编写，macOS下推荐使用[William Kyngesburye](http://www.kyngchaos.com/software:frameworks)维护的Framework安装包，最新的为`GDAL_Complete-1.11.dmg`。
52 
53 + GDAL将安装在`/Library/Frameworks/GDAL.framework`。
54 + 同时，测试python的GDAL包是否安装成功：
55 
56  ```python
57  >>> import osgeo
58  >>> osgeo.__version__
59  '1.11.4'
60  >>> from osgeo import ogr
61  >>> from osgeo import osr
62  >>> from osgeo import gdalconst
63  >>> from osgeo import gdal_array
64  >>> from osgeo import gdal
65  ```
66 + 此外，还应将GDAL自带可执行程序的安装目录添加至环境变量PATH中，以便可以直接在Terminal中调用GDAL程序：
67 参考[这篇博客](http://architectryan.com/2012/10/02/add-to-the-path-on-mac-os-x-mountain-lion/#.WV47mtOGOu5)
68  ```bash
69  ~ sudo nano /etc/paths
70  # Enter your password, when prompted.
71  # Go to the bottom of the file, and enter the path you wish to add.
72  /Library/Frameworks/GDAL.framework/Programs
73  # Hit control-x to quit.
74  # Enter “Y” to save the modified buffer.
75  # To test it, in new terminal window, type:
76  ~ echo $PATH
77  /usr/local/bin:/usr/bin:/bin:/usr/sbin:/sbin:/Library/Frameworks/GDAL.framework/Programs
78  # 测试
79  ~ gdalinfo --version
80 
81  GDAL 1.11.4, released 2016/01/25
82  ```

84 ## 1.5. mongo-c-driver
85 
86 SEIMS数据管理采用NoSQL型数据库——MongoDB，依赖于mongo-c-driver。
87 macOS下的配置步骤为：
88 + 从[官网](http://mongoc.org/ "mongo-c-driver-download")下载源码压缩包，目前最新稳定版本为[1.5.0](https://github.com/mongodb/mongo-c-driver/releases/download/1.5.0/mongo-c-driver-1.5.0.tar.gz "mongo-c-driver-1.5.0")，解压缩至当前文件夹，如`/Users/zhulj/apps/mongo-c-driver-1.5.0
89 
90 + 使用brew安装依赖包`brew install automake autoconf libtool pkgconfig openssl`
91 + 打开终端，依次输入如下命令
92 
93  ```shell
94  cd /Users/zhulj/apps/mongo-c-driver-1.5.0
95  export LDFLAGS="-L/usr/local/opt/openssl/lib"
96  export CPPFLAGS="-I/usr/local/opt/openssl/include"
97  ./configure CC=gcc-4.9 CXX=g++-4.9 --prefix=/usr/local
98  make
99  sudo make install
100  ```
101 
102 
103 + 至此，`mongo-c-driver`即编译安装完成了，在`/usr/local/include`目录下能看到`libbson-1.0`, `libmongoc-1.0`文件夹，链接库则在`/usr/local/lib`。

105 # 2. Compilation and Installation
106 
107 使用CMake编译SEIMS时有3个可选参数：
108 1. `-DPARALLEL`，不添加则默认为编译`OpenMP`版本程序，添加`-DPARALLEL=MPI`则编译MPI/OpenMP混合版本；
109 2. `-DARCH`，~~用于指定编译32位还是64程序(目前仅适用于Windows)，需要与`GDAL`和`mongo-c-driver`版本匹配，不添加则默认为32位程序，添加`-DARCH=64`则为64位；~~
110 3. `-DSTROM`，用于指定是否编译次降水模型，不添加默认为0，即长时段模型，添加`-DSTROM=1`则编译次降水模型。
111 4. `-DUNITTEST`，用于指定是否编译基于Googletest的单元测试模块
112 5. `-DINSTALL_PREFIX`，用于指定安装路径，如不指定，默认为源码目录下的`bin`文件夹。

114 ## 2.1 Installation for users
115 
116 ```shell
117 cd /Users/zhulj/Documents/code/SEIMS/seims
118 mkdir build
119 cd build
120 1. clang: cmake .. -DCMAKE_BUILD_TYPE=Release
121 2. GCC: cmake .. -DCMAKE_BUILD_TYPE=Release -DCMAKE_C_COMPILER=/usr/local/bin/gcc-4.9 -DCMAKE_CXX_COMPILER=/usr/local/bin/g++-4.9
122 make
123 make install
124 ```
125 + 在执行`make`命令时，可追加 `-j 4`命令利用CPU多核性能并行编译程序，提高编译速度，并行进程数最好设置为物理核数的2倍。
126 + 编译、安装无误后，SEIMS所有可执行程序及模块动态链接库均在`./SEIMS/seims/bin`。
127 
128 + 注意1：使用clang编译时，由于其不支持openmp，因此基于共享内存的并行计算将无法实现，因此推荐使用GCC进行编译安装。

130 ## 2.2 Installation for developers
131 
132 ### 2.2.1 Xcode (test on Xcode 8.2)
133 
134 ```shell
135 cd /Users/zhulj/Documents/code/SEIMS/seims
136 mkdir build
137 cd build
138 cmake -G "Xcode" .. -DCMAKE_BUILD_TYPE=Release
139 ```
140 + 完成之后，即可得到SEIMS项目总工程，`SEIMS_OMP_ALL.xcodeproj`，各子工程则在`build/bin`下，如SEIMS模块库工程为`build/bin/seims_omp_project/SEIMS_OMP_prj.xcodeproj`。
141 
142 ### 2.2.2 CLion (test on CLion 2016.3.2）
143 
144 CLion继承了JetBrains家族的特点，工程采用CMake管理，方便、高效。在CLion中直接打开`SEIMS/seims`文件夹即可导入整个工程。
145 
146 CLion默认使用clang编译器，因此需要简单设置一下，[官网提供了三种方法](https://cmake.org/Wiki/CMake_FAQ#How_do_I_use_a_different_compiler.3F)，推荐采用第二种，即：
147 
148 + 打开CLion->Preferences->Build，Execution，Deployment->CMake
149 + 在Generation选项卡下的CMake options中输入以下命令，以覆盖CMake的默认设置，同理为Debug和Release分别设置。
150 
151 ```
152 -D CMAKE_C_COMPILER=<your_path_to_c_compiler>
153 -D CMAKE_CXX_COMPILER=<your_path_to_cxx_compiler>
154 
155 e.g.
156 -DCMAKE_C_COMPILER=/usr/local/bin/gcc-4.9 -DCMAKE_CXX_COMPILER=/usr/local/bin/g++-4.9
157 ```
158 
159 接下来，你可以选择Build All，也可以选择某一个程序进行Build等。

161 ## 3. Python for SEIMS 依赖包的安装
162 
163 SEIMS预处理、后处理、率定、情景分析等所需的Python依赖库及版本要求如下：
164 
165 ```py
166 pygeoc
167 # Preprocess
168 GDAL>=1.9.0,<2.0 (在1.4中已经安装完成)
169 numpy>=1.9.0
170 pymongo>=3.0
171 networkx>=1.10
172 Shapely>=1.5.0
173 # Postprocess
174 matplotlib>=1.5.0
175 pathlib2>=2.0.0
176 # Scenario analysis
177 deap>=1.0.2
178 scoop>=0.7.1.1
179 ```
180 
181 + SEIMS提供了自动安装脚本，脚本依赖于pip，因此请确保pip已正确安装，[参考教程](https://pip.pypa.io/en/stable/installing/)：
182  + 下载[get-pip.py](https://bootstrap.pypa.io/get-pip.py)
183  + `python get-pip.py`
184 
185 + 执行`sudo sh pyseims_install_linux.sh`即可自动安装完成所有依赖库，包括SEIMS自带的`PyGeoC`。
186 
187 + 安装完成后，运行`python pyseims_check.py`检查所需python库是否已经安装成功，如果有未成功的请单独安装。
