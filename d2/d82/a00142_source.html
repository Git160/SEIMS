14 ## 2 Install mongo-c-driver
15 
16 ### 2.1 On Windows
17 + Windows开发环境推荐采用Microsoft Visual Studio 2010 或更高
18 + 从[官网](http://mongoc.org/ "mongo-c-driver-download")下载源码压缩包，目前最新稳定版本为[1.5.0](https://github.com/mongodb/mongo-c-driver/releases/download/1.5.0/mongo-c-driver-1.5.0.tar.gz "mongo-c-driver-1.5.0")，解压缩至当前文件夹，如`C:\z_code\Repos\mongo-c-driver-1.5.0`
19 + 打开cmd，依次输入如下命令，默认的安装目录为`C:\mongo-c-driver`
20 
21  ```bat
22  cd C:\
23  mkdir mongo-c-driver
24  cd C:\z_code\Repos\mongo-c-driver-1.5.0
25  cd src\libbson
26  cmake -DCMAKE_INSTALL_PREFIX=C:\mongo-c-driver -G "Visual Studio 10 2010"
27  msbuild.exe ALL_BUILD.vcxproj
28  msbuild.exe INSTALL.vcxproj
29  cd ..\..
30  cmake -DCMAKE_INSTALL_PREFIX=C:\mongo-c-driver -DBSON_ROOT_DIR=C:\mongo-c-driver -G "Visual Studio 10 2010"
31  msbuild.exe ALL_BUILD.vcxproj
32  msbuild.exe INSTALL.vcxproj
33  ```
34 
35 + 至此，`mongo-c-driver`即编译安装完成了，在`C:\mongo-c-driver`目录下能看到`bin`, `include`, `lib`文件夹。
36 + 将`C:\mongo-c-driver`添加至环境变量，命名为`MONGOC_ROOT_DIR`。
37 + 将`C:\mongo-c-driver\bin`添加至环境变量，命名为`MONGOC_LIB_DIR`，并将`%MONGOC_LIB_DIR%`添加至环境变量`PATH`中。
38 
39 > Note: 如果cmd提示找不到msbuild.exe，可以在msbuild.exe前加上绝对路径，这个文件是.NetFramework里的，比如`C:\Windows\Microsoft.NET\Framework64\v4.0.30319\msbuild.exe`

41 ### 2.2 On Linux
42 
43 > 以CentOS为例 (Take CentOS as example)
44 
45 + 下载源码包，如`mongo-c-driver-1.6.0.tar.gz`
46 + 解压、编译、安装，命令如下：
47 
48  ```shell
49  tar xzf mongo-c-driver-1.6.0.tar.gz
50  cd mongo-c-driver-1.6.0
51  ./configure --disable-automatic-init-and-cleanup
52  make
53  make install
54  ```
55 
56 + 值得注意的是，在`./configure`时可以修改一些参数，比如GCC版本、安装目录等，比如我的配置为：`./configure --prefix=~/code/mongo-c-driver CC=~/gcc4.8.4/bin/gcc CXX=~/gcc4.8.4/bin/g++ --disable-automatic-init-and-cleanup`
57 + 安装完成之后，`bin`、`include`、`lib`等目录均在`/usr/local`或自定义的安装目录下，
58 + 在`/etc/profile`（对所有用户有效）或`~/.bash_profile`（对当前用户有效）内添加环境变量以及追加动态链接库目录
59 
60  ```shell
61  e.g.
62 
63  export MONGOC_ROOT_DIR=~/code/mongoc-c-driver
64  export PKG_CONFIG_PATH=~/code/mongoc-c-driver/lib/pkgconfig:$PKG_CONFIG_PATH
65  export LD_LIBRARY_PATH=~/code/mongoc-c-driver/lib:$LD_LIBRARY_PATH
66  ```
67 
68 + 保存后，通过`source /etc/profile`或`source ~/.bash_profile`命令使设置生效
69

71 ### 2.3 On macOS
72 + 从[官网](http://mongoc.org/ "mongo-c-driver-download")下载源码压缩包，目前最新稳定版本为[1.5.0](https://github.com/mongodb/mongo-c-driver/releases/download/1.5.0/mongo-c-driver-1.5.0.tar.gz "mongo-c-driver-1.5.0")，解压缩至当前文件夹，如`/Users/zhulj/apps/mongo-c-driver-1.5.0`
73 + 使用brew安装依赖包`brew install automake autoconf libtool pkgconfig openssl`
74 + 打开终端，依次输入如下命令
75  ```shell
76  cd /Users/zhulj/apps/mongo-c-driver-1.5.0
77  export LDFLAGS="-L/usr/local/opt/openssl/lib"
78  export CPPFLAGS="-I/usr/local/opt/openssl/include"
79  ./configure CC=gcc-4.9 CXX=g++-4.9 --prefix=/usr/local
80  make
81  sudo make install
82  ```
83 
84 + 至此，`mongo-c-driver`即编译安装完成了，在`/usr/local/include`目录下能看到`libbson-1.0`, `libmongoc-1.0`文件夹，链接库则在`/usr/local/lib`。

86 ## 3 MongoUtilClass测试
87 + MongoUtilClass采用CMake进行跨平台编译。
88 + MongoUtilClass需调用[UtilsClass](https://github.com/lreis2415/UtilsClass)，编译前需将其保存至MongoUtilClass**同级目录**下。
89 
90 ### 3.1 Windows
91 + 打开 “开始” -> Microsoft Visual Studio 2010 -> Visual Studio Tools -> Visual Studio 命令提示(2010)，以**管理员方式**运行，依次输入以下命令：
92 
93  ```shell
94  cd <path-to-MongoUtilClass>
95  mkdir build
96  cd build
97  ### 仅编译安装 ###
98  cmake -G "NMake Makefiles" <path-to-MongoUtilClass> -DCMAKE_BUILD_TYPE=Release
99  nmake
100  nmake install
101  ### 编译Microsoft Visual Studio工程 ###
102  cmake <path-to-MongoUtilClass>
103  nmake
104  ```
105 
106 + 对于“仅编译安装”操作，`MongoUtilClass.exe`会自动安装在`<path-to-MongoUtilClass>\bin`目录下。
107 + 对于“编译Microsoft Visual Studio工程”，`MongoUtil.sln`将保存在`<path-to-MongoUtilClass>\build`目录下。
108 
109 ### 3.2 Linux
110 
111  ```shell
112  cd MongoUtilClass
113  mkdir build
114  cd build
115  cmake ..
116  make
117  make install
118  ```
119 + 可执行程序`MongoUtil`默认安装于`MongoUtilClass/bin`下
120 + 如果希望采用其他GCC安装版本，请在cmake命令后面追加：`-DCMAKE_C_COMPILER=~/gcc4.8.4/bin/gcc -DCMAKE_CXX_COMPILER=~/gcc4.8.4/bin/g++`
