35 # 1. Prerequisites
36 
37 SEIMS模型采用C++和Python编写，支持子流域-栅格双层并行计算，在Windows下推荐采用Microsoft Visual Studio进行编译安装，安装前的准备包括：
38 + Microsoft Visual Studio 2010 或更高 -- C++源码编译器及IDE
39 + CMake 3.1+ -- 管理代码编译
40 + GDAL 1.x with Python -- 矢栅数据读写库
41 + mongo-c-driver 1.5+ -- 读写MongoDB
42 + MS-MPI v6 （或更高）-- 编译MPI并行程序，如TauDEM、SEIMS-MPI版本等
43 
44 ## 1.1 Microsoft Visual Studio
45 
46 + **For User**： 如果只是希望源码编译SEIMS模型，而不想安装臃肿庞大的VS，可以选择使用Microsoft Visual C++ Build Tools，比如[2015 Update 3](https://www.visualstudio.com/downloads/#microsoft-visual-c-build-tools-2015-update-3 "microsoft-visual-c-build-tools-2015-update-3").
47 + **For Developer**： 如果希望对SEIMS模型进行改进，建议采用[Microsoft Visual Studio 2010 或更高版本](https://www.visualstudio.com/downloads/ "visual-studio-downloads")进行开发。
48 
49 > Note: 虽然SEIMS编译理论上支持更高版本的VS，但是为了接下来的GDAL库安装方便，建议使用与[Tamas Szekeres's Windows GDAL binaries](http://www.gisinternals.com/release.php)发布版本对应的VS，比如目前其发布的GDAL安装版本对应的最高VS版本为2013。当然，如果你希望自己源码编译接下来用到的GDAL、mongo-c-driver库，那VS版本选择则不受限制。
50 
51 ## 1.2. CMake
52 
53 CMake是一个跨平台的安装或编译工具，可以用简单的语句来描述安装或编译过程。CMake通过`CmakeLists.txt`文件能够输出各种各样的makefile或者IDE工程。
54 
55 CMake可以从其[官网免费下载](http://www.cmake.org/files)，最低安装版本要求为3.1，安装后添加CMake路径如`C:\Program Files (x86)\CMake`到系统环境变量`PATH`里。
56 
57 ## 1.3. GDAL and Python
58 
59 SEIMS的矢栅数据读写基于`GDAL 1.x`编写，Windows下GDAL最方便的安装方式应该就是采用[Tamas Szekeres's Windows GDAL binaries](http://www.gisinternals.com/release.php)编译发布的版本，下载时请注意区分32位或64位版本。
60 
61 + 具体安装请参考博客[Installing-gdal-with-python-for-windows](https://sandbox.idre.ucla.edu/sandbox/tutorials/installing-gdal-for-windows "installing-gdal-with-python-for-windows")。
62 
63 + **注意**：安装完成之后，除了`GDAL_DATA`外，还需要在环境变量里添加一个`GDAL_DIR`，赋值为你的GDAL安装目录，如`C:\GDAL`，以便CMake在编译时能够找得到GDAL依赖库。
64 
65 + **注意**：为了方便将来程序能够找到GDAL的诸多动态链接库，建议新建`GDAL_PATHS`环境变量`C:\GDAL;C:\GDAL\bin;C:\GDAL\bin\proj\apps;C:\GDAL\bin\gdal\apps;C:\GDAL\bin\ms\apps;C:\GDAL\bin\curl;`，并将`%GDAL_PATHS%`添加至环境变量`PATH`中。
66 
67 ## 1.4. mongo-c-driver
68 
69 SEIMS数据管理采用NoSQL型数据库——MongoDB，依赖于mongo-c-driver。
70 Windows下的配置步骤为：
71 + 从[官网](http://mongoc.org/ "mongo-c-driver-download")或[Github项目](https://github.com/mongodb/mongo-c-driver/releases)下载源码压缩包，如[1.5.0](https://github.com/mongodb/mongo-c-driver/releases/download/1.5.0/mongo-c-driver-1.5.0.tar.gz "mongo-c-driver-1.5.0")，解压缩至当前文件夹，如`C:\z_code\Repos\mongo-c-driver-1.5.0`
72 + 打开cmd，依次输入如下命令，默认的安装目录为`C:\mongo-c-driver`
73 
74  ```bat
75  cd C:\
76  mkdir mongo-c-driver
77  cd C:\z_code\Repos\mongo-c-driver-1.5.0
78  cd src\libbson
79  cmake -DCMAKE_INSTALL_PREFIX=C:\mongo-c-driver -G "Visual Studio 10 2010"
80  msbuild.exe ALL_BUILD.vcxproj
81  msbuild.exe INSTALL.vcxproj
82  cd ..\..
83  cmake -DCMAKE_INSTALL_PREFIX=C:\mongo-c-driver -DBSON_ROOT_DIR=C:\mongo-c-driver -G "Visual Studio 10 2010"
84  msbuild.exe ALL_BUILD.vcxproj
85  msbuild.exe INSTALL.vcxproj
86  ```
87 
88 + 至此，`mongo-c-driver`即编译安装完成了，在`C:\mongo-c-driver`目录下能看到`bin`, `include`, `lib`文件夹。
89 + 将`C:\mongo-c-driver`添加至环境变量，命名为`MONGOC_ROOT_DIR`。
90 + 将`C:\mongo-c-driver\bin`添加至环境变量，命名为`MONGOC_LIB_DIR`，并将`%MONGOC_LIB_DIR%`添加至环境变量`PATH`中。
91 
92 > Note1: 如果cmd提示找不到msbuild.exe，可以在msbuild.exe前加上绝对路径，这个文件是.NetFramework里的，比如`C:\Windows\Microsoft.NET\Framework64\v4.0.30319\msbuild.exe`
93 
94 > Note2: 如果想编译64位的mongo-c-driver，则需将上述代码中的`-G "Visual Studio 10 2010"`替换为`-G "Visual Studio 10 2010 Win64"`
95 
96 ## 1.5. MS-MPI
97 
98 Microsoft MPI (MS-MPI) 是微软基于MPICH实现的用于Windows平台开发和运行并行应用程序的消息传递接口标准，预处理程序中用于提取子流域和地形信息的TauDEM、SEIMS_MPI并行版本等都需要MPI的支持。
99 
100 + 从[MS-MPI v6](https://www.microsoft.com/en-us/download/details.aspx?id=47259)下载并分别安装msmpisdk.msi, MSMpiSetup.exe，并检查系统环境变量是否配置正确，如Windows-64bit版：
101 
102  ```
103  MSMPI_BIN=C:\Program Files\Microsoft MPI\Bin\
104  MSMPI_INC=C:\Program Files (x86)\Microsoft SDKs\MPI\Include\
105  MSMPI_LIB32=C:\Program Files (x86)\Microsoft SDKs\MPI\Lib\x86\
106  MSMPI_LIB64=C:\Program Files (x86)\Microsoft SDKs\MPI\Lib\x64\
107  ```
108 
109 > Note: 建议采用默认安装路径安装MPI

111 # 2. Compilation and Installation
112 
113 使用CMake编译SEIMS时有3个可选参数：
114 1. `-DPARALLEL`，不添加则默认为编译`OpenMP`版本程序，添加`-DPARALLEL=MPI`则编译MPI/OpenMP混合版本；
115 2. `-DARCH`，用于指定编译32位还是64程序，需要与`GDAL`和`mongo-c-driver`版本匹配，不添加则默认为32位程序，添加`-DARCH=64`则为64位；
116 3. `-DSTROM`，用于指定是否编译次降水模型，不添加默认为0，即长时段模型，添加`-DSTROM=1`则编译次降水模型。
117 4. `-DUNITTEST`，用于指定是否编译基于Googletest的单元测试模块
118 5. `-DINSTALL_PREFIX`，用于指定安装路径，如不指定，默认为源码目录下的`bin`文件夹。
119 
120 ## 2.1 Installation for users
121 
122 > Note: Windows x64下，不要打开的是`Visual Studio x64 Win64 命令提示(2010)`，而是需要打开`Visual Studio 命令提示(2010)`
123 
124 + 打开 “开始” -> Microsoft Visual Studio 2010 -> Visual Studio Tools -> Visual Studio 命令提示(2010)，以**管理员方式**运行，依次输入以下命令：
125 
126  ```shell
127  cd ~\SEIMS\seims
128  mkdir build
129  cd build
130  cmake -G "NMake Makefiles" .. -DCMAKE_BUILD_TYPE=Release
131  nmake
132  nmake install
133  ```
134 
135 + 或者采用指定VS版本及位数的方式：
136 
137  ```shell
138  cd ~\SEIMS\seims
139  mkdir build
140  cd build
141  # 编译32位版本，-DARCH=32 为默认，添加与否均可
142  cmake -G "Visual Studio 10 2010" ..
143  # 编译64位版本，此时一定确保GDAL和mongo-c-driver已配置好64位版本
144  cmake -G "Visual Studio 10 2010 Win64" .. -DARCH=64
145  msbuild.exe ALL_BUILD.vcxproj
146  # p.s.1 如果编译release版本：
147  # msbuild.exe ALL_BUILD.vcxproj /p:Configuration=Release
148  # p.s.2 可通过添加/maxcpucount:4 来并行编译，提高速度
149  msbuild.exe INSTALL.vcxproj
150  # 如果编译release版本：
151  # msbuild.exe INSTALL.vcxproj /p:Configuration=Release
152  ```
153 
154 + 完成后，不需要打开VS2010生成解决方案，可执行文件已经编译完成，程序目录为`~\SEIMS\seims\bin`或通过`-DINSTALL_PREFIX`指定的文件夹。
155 + 右键以管理员方式运行`~\seims\bin\Firewall_for_Windows.bat`，配置防火墙规则，以防在运行Python脚本调用SEIMS程序时被防火墙阻止。
156 ## 2.2 Installation for developers
157 
158 + 打开 “开始” -> Microsoft Visual Studio 2010 -> Visual Studio Tools -> Visual Studio 命令提示(2010)，以**管理员方式**运行，依次输入以下命令：
159 
160  ```shell
161  cd ~\SEIMS----------7\seims
162  mkdir build
163  cd build
164  cmake ..
165  ```
166 
167 + 然后用VS2010打开< build >文件夹下的`SEIMS_OMP_ALL.sln`，即为整个SEIMS模型的VS工程，打开`build\bin`目录可以分别找到子工程，比如SEIMS模块库工程等。
168 + VS工程编译完成之后，右键单击`INSTALL`模块，选择“生成”，即可完成SEIMS模型的安装，安装路径为`~\SEIMS\seims\bin`
169 + 安装完成SEIMS后，需要右键以管理员方式运行`~\SEIMS\seims\Firewall_for_Windows.bat`配置防火墙规则，以防在运行Python脚本调用SEIMS程序时被防火墙阻止。

173 ## 3. Python for SEIMS 依赖库的安装
174 
175 SEIMS预处理、后处理、率定、情景分析等所需的Python依赖库及版本要求如下：
176 
177 ```python
178 pygeoc
179 # Preprocess
180 GDAL>=1.9.0,<2.0
181 numpy>=1.9.0
182 pymongo>=3.0
183 networkx>=1.10
184 Shapely>=1.5.0
185 # Postprocess
186 matplotlib>=1.5.0
187 pathlib2>=2.0.0
188 # Scenario analysis
189 deap>=1.0.2
190 scoop>=0.7.1.1
191 ```
192 
193 + SEIMS提供了自动安装脚本，脚本依赖于pip，因此请确保pip已正确安装，[参考教程](https://pip.pypa.io/en/stable/installing/)：
194  + 下载[get-pip.py](https://bootstrap.pypa.io/get-pip.py)
195  + `python get-pip.py`
196 
197 + 以管理员方式运行`pyseims_install_windows.bat`即可自动安装完成所有依赖库，包括SEIMS自带的`PyGeoC`。
198 
199 + 安装完成后，运行`python pyseims_check.py`检查所需python库是否已经安装成功，如果有未成功的请单独安装。
