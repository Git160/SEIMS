14 ##### 1.1.2 气象数据
15 由于研究区周围仅有一个国家气象站点，合肥站（58321），因此采用这一个站点的日值观测数据。数据来源为[中国气象数据网](http://data.cma.cn/ "中国气象数据网")的中国地面气候资料日值数据集(V3.0)产品，[点此查看数据下载方法](http://zhulj.net/python/2016/04/11/Constructing-SURF_CLI_CHN_MUL_DAY_V3.0-database.html)。

19 对于日尺度模型的构建，所需气象数据如下：
20 + 气象数据(**日值**)
21  + StationID: 必需，站点编号(整型)
22  + Y, M, D : 必需，年/月/日
23  + TMAX : 必需，最高气温(deg C)
24  + TMIN : 必需，最低气温(deg C)
25  + TMEAN : 可选，平均气温(deg C)，缺失情况下，由(TMAX+TMIN)/2计算
26  + RM : 必需，相对湿度(%)
27  + WS : 必需，风速(m/s)
28  + SR/SSD : 二选一，太阳辐射(MJ/m2/day)/日照时数(h)
29  + PET : 可选，蒸散发(mm)

93 #### 3.3 管理措施情景设置
94 
95 #### 3.3.1 输入文件组织
96 
97 + BMP_index.txt：SEIMS模型支持的BMP类型编号 （目前多数尚未实现）
98 + BMP_scenarios.txt：情景分析设置的主文件，包含以下属性列：
99  + ID：唯一标识情景ID
100  + NAME：情景名称
101  + BMPID：BMP类型，如12为作物管理，1为点源污染
102  + SUBSCENARIO：子情景唯一标识ID，某一BMP类型可能存在多个子情景，如作物管理下可分为0传统施肥方式和1生态种植技术
103  + DISTRIBUTION：定义该BMP的分布，多个字段用‘|’分割，第一个可为RASTER和ARRAY，分别意为栅格数据和列表；如为栅格数据，则第二个为栅格名（如MGT_FIELDS），如果为列表，第二个则为MongoDB表明，其中存储点状要素列表（如point_source_distribution）；如果为列表，第三个数为点状要素类别（如10000为养牛厂）
104  + COLLECTION：管理措施参数表，如点源污染措施表为point_source_management
105  + LOCATION：用于定义该措施实施位置，如果DISTRIBUTION中为RASTER，则LOCATION定义栅格值，多个值之间用‘，’分割，如“1,3,4”，如果作用于整个流域，则输入“ALL”；同理，如果DISTRIBUTION为ARRAY，LOCATION中输入COLLECTION中定义的点编号，如“20001,20003”等
106 + point_source_distribution.txt：如上所述，定义点源污染位置，包括
107  + PTSRC：点源类型，包括养牛厂（10000），养猪厂（20000），上游水库放水点（30000），居民生活污水排放口（40000）
108  + PTSRCID：具体点源编号，支持1~9999
109  + Name：点源名称
110  + Lon，Lat：经纬度
111  + LocalX，LocalY：投影坐标值，用于判断该点源所在子流域
112  + Size：规模，用于计算排放总量，如10001养牛厂养殖300头牛
113 + point_source_management.txt：点源管理，用于确定排放量及排放日期等，详见该文件头中注释（#开头）
114 + areal_source_distribution.txt 与 areal_source_management.txt 同上，不再赘述。
115 + plant_management.txt：作物管理措施，根据积温或者种植日期，管理作物生长，输入参数借鉴SWAT。
