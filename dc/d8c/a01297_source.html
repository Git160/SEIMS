12 
13 SEIMS数据预处理，即将原始输入数据、参数等进行初步加工并导入MongoDB数据库，以供SEIMS运行时调用。
14 
15 下面，介绍如何配置预处理程序完成SEIMS数据预处理任务。
16 
17 ## 1. config.py
18 
19 SEIMS预处理程序的所有输入均在`SEIMS/preprocess/config.py`中进行设置。
20 
21 ### 1.1 模型相关路径设置
22 
23 程序的第10~15行或17~22行：
24 
25 + BASE_DATA_DIR: 准备数据所在目录，里面应该包含`climate`和`spatial`文件夹及相关数据
26 + TXT_DB_DIR: SEIMS文本类数据库目录，如各类查找表
27 + PREPROC_SCRIPT_DIR: SEIMS预处理Python脚本目录
28 + CPP_PROGRAM_DIR: SEIMS预处理相关C++程序编译目录
29 + MPIEXEC_DIR: MPI程序目录，如环境变量里有设置，则此项可为`None`
30 
31 以下为一个示例：
32 
33 ```python
34 BASE_DATA_DIR = r'E:\data\model_data\model_dianbu_30m_longterm\data_prepare'
35 TXT_DB_DIR = r'E:\code\Hydro\SEIMS\database'
36 PREPROC_SCRIPT_DIR = r'E:\code\Hydro\SEIMS\preprocess'
37 CPP_PROGRAM_DIR = r'D:\Compile\SEIMS_Preprocess\Release'
38 MPIEXEC_DIR = None
39 ```
40 
41 ### 1.2 MongoDB相关设置
42 
43 + HOSTNAME: MongoDB的IP地址，默认为本机`127.0.0.1`
44 + PORT: MongoDB端口号，默认为27017
45 + ClimateDBName: 气象数据库名
46 + SpatialDBName: 主数据库名，即模型名
47 + forCluster: `True` 或 `False`，用于标识是否用于SEIMS_MPI版本，默认为`False`
48 + stormMode: 是否用于次暴雨建模，默认为`False`
49 
50 > `forCluster` 如为`True`，则说明所建数据库是为`SEIMS_MPI`并行版本准备，此时，所有栅格数据均按照子流域编号进行分割后分别存储，如坡度数据在`forCluster`为`False`时，数据库中存储为`1_SLOPE`,而`forCluster`为True时，则存储为`1_SLOPE`, `2_SLOPE`...`N_SLOPE`
51 
52 以下为一个示例：
53 
54 ```python
55 HOSTNAME = '127.0.0.1'
56 PORT = 27017
57 ClimateDBName = 'climate_dianbu'
58 SpatialDBName = 'model_dianbu_30m_longterm'
59 forCluster = False
60 stormMode = False
61 ```
62 
63 ### 1.3 气象数据输入
64 
65 + HydroClimateVarFile: 气象数据单位定义
66 + MetroSiteFile: 气象站点信息
67 + PrecSiteFile: 降水站点信息
68 + MeteoDailyFile: 气象数据
69 + PrecExcelPrefix: 降水数据前缀
70 + PrecDataYear: 降水数据年份，为数组，多个年份应该对应多个降水EXCEL文件
71 
72 以下为一个示例:
73 
74 ```python
75 HydroClimateVarFile = CLIMATE_DATA_DIR + os.sep + 'Variables.txt'
76 MetroSiteFile = CLIMATE_DATA_DIR + os.sep + 'Sites_M.txt'
77 PrecSiteFile = CLIMATE_DATA_DIR + os.sep + 'Sites_P.txt'
78 MeteoDailyFile = CLIMATE_DATA_DIR + os.sep+ 'meteorology_dianbu_daily.txt'
79 PrecExcelPrefix = CLIMATE_DATA_DIR + os.sep + 'precipitation_by_day_'
80 PrecDataYear = [2014]
81 ```
82 
83 > TODO: 降水数据的输入应改为以站点编号（或站名）命名的文本文件，而非以年份组织的EXCEL
84 
85 ### 1.4 空间数据输入
86 
87 + PrecSitesThiessen: 降水站点泰森多边形分布
88 + MeteorSitesThiessen: 气象站点泰森多边形分布
89 + dem: DEM数据
90 + outlet_file:流域出口，可设置为`None`
91 + threshold: 河网提取汇流累积量阈值，默认为0，则采用TauDEM算法[DropAnalysis](http://hydrology.usu.edu/taudem/taudem5/help53/StreamDropAnalysis.html)自动确定
92 + np: TauDEM运行时并行计算进程数，默认为4
93 + landuseFile: 土地利用数据
94 + soilSEQNFile: 土壤类别序列数据
95 + soilSEQNText: 土壤类别序列属性查找表
96 
97 以下为一个示例:
98 
99 ```python
100 PrecSitesThiessen = SPATIAL_DATA_DIR + os.sep + 'Preci_dianbu_Vor.shp'
101 MeteorSitesThiessen = SPATIAL_DATA_DIR + os.sep + 'Metero_hefei_Vor.shp'
102 dem = SPATIAL_DATA_DIR + os.sep + 'dem_30m.tif'
103 outlet_file = SPATIAL_DATA_DIR + os.sep + 'outlet_30m.shp'
104 threshold = 0
105 np = 4 # number of parallel processors for TauDEM
106 landuseFile = SPATIAL_DATA_DIR + os.sep + 'landuse_30m.tif'
107 soilSEQNFile = SPATIAL_DATA_DIR + os.sep + 'soil_SEQN.tif'
108 soilSEQNText = SPATIAL_DATA_DIR + os.sep + 'soil_properties_lookup.txt'
109 ```

125 ## 4. 预处理子程序调用
126 
127 多数情况下，模型的预处理不可能一次成功，往往需要多次修改，而并非每次修改都需要运行整个预处理流程，因此，SEIMS提供了预处理过程中的子程序的调用。
128 
129 基本调用命令与之前一致，均为：
130 
131 ```python
132 > python XXX.py -ini <configurationFile>.ini
133 ```
